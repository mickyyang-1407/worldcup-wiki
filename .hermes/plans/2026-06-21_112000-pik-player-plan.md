## 技術棧（已拍板）

| 層級 | 技術 | 原因 |
|------|------|------|
| 框架 | **Tauri 2.0** (Rust) | 不捆 Chromium，記憶體低，macOS 原生 |
| 前端 | **Solid.js + TypeScript** | 輕量、reactivity 強，少 overhead |
| 播放引擎（一般音訊） | **mpv** (via `brew install mpv` + Rust `mpv` crate) | 所有格式 + gapless |
| 播放引擎（Atmos MP4） | **Apple AVFoundation** (via `objc2-avfoundation` Rust crate) | macOS 原生 Spatial Audio，不需跳 QuickTime |
| 資料庫 | **SQLite** (`rusqlite`, bundled) | 輕量免 server |
| 檔案監視 | **notify** crate | 即時音樂庫更新 |
| CSS | **Tailwind CSS** | 快速開發 |
| 打包 | **Tauri bundler** → `.dmg` | macOS 專用，M 晶片原生 |

## 已確認的決策

### ✅ Atmos MP4 播放 — 嵌入式 AVFoundation

| 檔案類型 | 播放引擎 | 方式 |
|---------|---------|------|
| FLAC / WAV / MP3 / AAC / OGG / M4A (無 Atmos) | **mpv** | 內嵌播放（Tauri 視窗內） |
| MP4 含 Dolby Atmos | **AVFoundation** (via `objc2-avfoundation`) | 同視窗內 AVPlayer 播放 |
| MKV / 其他 | **mpv** | 內嵌播放 |

**Atmos 偵測：** 播放前先用 FFmpeg probe 判斷 MP4 是否含 E-AC3 JOC / Atmos 元資料
- `ffprobe -v quiet -select_streams a:0 -show_entries stream=codec_name,codec_tag_string -of json <file>`
- 若 `codec_name` = `eac3` 且 codec_tag 含 Atmos 標記 → 是 Atmos
- 無 Atmos → mpv
- 有 Atmos → Rust 端用 `objc2-avfoundation` 產生 AVPlayer 實例播放

### ✅ UI 風格

- 不要全黑、不要 Apple Music clone
- 有自己的風格，混合參考多個來源
- 乾淨、專業但不沈悶

### ✅ Mini Player

參考 Apple Music mini player 的比例概念，但有自己的視覺語言

### ✅ 跨平台

- **macOS only** — M 晶片原生
- 保留 Tauri 跨平台可能性，但第一版不支援 Windows/Linux

### ✅ GitHub

放 Micky 個人帳號下，repo name: `pik-player`

## 階段性實施計劃

### Phase 0 — Project Scaffold

- `npm create tauri-app` → Solid.js + TypeScript
- Tauri 2.0 初始化（macOS bundle identifier: `com.mickydigitalstudio.pikplayer`）
- 安裝 Rust 依賴：`rusqlite` (bundled), `notify`, `mpv`, `lofty`, `objc2-avfoundation`
- 前端依賴：Tailwind CSS, Solid.js router
- Git repo init + MIT license + README

### Phase 1 — 核心播放引擎（mpv）

- Rust `mpv` crate 初始化 + 事件迴圈
- Tauri commands: `play`, `pause`, `toggle_play`, `seek`, `set_volume`, `load_file`, `get_position`, `get_duration`, `get_state`
- 支援格式：FLAC, WAV, MP3, OGG, M4A, AAC, MKV
- Gapless：`--gapless-audio=yes`
- 前端播放控件：進度條、時間顯示、音量滑桿、播放/暫停按鈕、上下首
- 鍵盤快速鍵：Space (播放/暫停), ← → (seek), ↑ ↓ (音量)

### Phase 2 — 音樂庫 & SQLite

- SQLite schema（tracks, artists, albums, playlists, playlist_tracks, playback_state）
- 掃描指定資料夾，遞迴搜尋音訊檔案
- `lofty` crate 讀取元資料（title, artist, album, track#...）
- 專輯封面提取與快取
- `notify` crate 監視檔案變動（新增/刪除/修改即時更新）
- 前端 Library 瀏覽：藝人 → 專輯 → 曲目（grid + list view）

### Phase 3 — 播放清單 & 斷點續播

- 建立/編輯/刪除播放清單（左側 sidebar）
- 拖曳排序（前端 drag & drop → Rust sync）
- 斷點續播：`playback_state` 表記錄播放位置
- 啟動時自動載入上次播放狀態 + 接續播放
- 每個播放清單獨立記憶最後播放曲目 + 秒數

### Phase 4 — 元資料編輯

- 編輯 title / artist / album / track number → `lofty` crate 寫回檔案
- 前端 inline edit（雙擊欄位直接修改）
- 確認修改後同步更新 SQLite

### Phase 5 — Mini Player & System Tray

- 按 **M** 切換 mini player 模式（縮小視窗）
- Mini player 佈局：封面縮圖（左）+ 歌名/藝人（中）+ 控制按鈕（右）
- 系統匣圖示：右鍵選單 — 播放/暫停、下一首、上一首、顯示主視窗、退出
- 關閉視窗 → 收進系統匣，不退出

### Phase 6 — 播放模式 & 進階功能

- 循環 / 單曲循環 / 隨機播放
- 即時搜尋（SQLite FTS5 或 LIKE query）
- 欄位排序（點擊欄位 header 排序：歌名、藝人、專輯、時間、播放次數）
- 播放次數統計 + 最近播放清單
- 拖曳外部檔案進視窗直接播放

### Phase 7 — Dolby Atmos MP4 支援 ⭐

- **Atmos 偵測：** 用 `ffprobe` 檢查 MP4 音訊串流是否有 E-AC3 JOC
- **一般 MP4（無 Atmos）：** 交給 mpv 播放
- **Atmos MP4：** 
  - Rust 端用 `objc2-avfoundation` crate 產生 AVPlayer 實例
  - 播放、暫停、seek、音量控制
  - 事件回傳：目前播放時間、播放結束、錯誤
  - 無縫與 mpv 引擎切換（前端 UI 統一控制，底層自動切引擎）

### Phase 8 — Polish & Package

- 自訂應用程式圖示
- macOS bundle：`.dmg`
- 鍵盤快速鍵完整支援
- README 撰寫（安裝方式、快捷鍵、支援格式）
- 初次啟動引導（選擇音樂資料夾）
